#    💰 DDS MANAGER | Учет движения денежных средств	 #
# ══════════════════════════════════════════════════════ #

Веб-приложение на Django для ведения учета корпоративных финансов. Система позволяет фиксировать доходы и расходы, структурировать их по категориям и анализировать денежные потоки с помощью удобных фильтров.


📦 Установка и запуск
	
	`python -m venv env`  			Создание виртуального окружения
	`source env/bin/activate`		Активация окружения (Linux/Mac)
	`env\Scripts\activate`			Активация окружения (Windows)
	`pip install -r requirements.txt`	Установка зависимостей
	`python manage.py migrate`		Применение миграций
	`python manage.py runserver`		**Запуск сервера**
	

🚀 Основные возможности

    	Ведение записей ДДС: 		Добавление, редактирование и удаление финансовых операций.
    	Иерархическая структура: 	Тип операции → Категория → Подкатегория.
    	Динамические формы: 		Автоматическое обновление списков категорий и подкатегорий при выборе типа операции (AJAX).
    	Фильтрация: 			Поиск и отбор записей по дате, статусу, типу и категориям.
    	Управление справочниками: 	Отдельные разделы для редактирования статусов, типов, категорий и подкатегорий.
    	Валидация данных: 		Защита от некорректного выбора категорий на уровне моделей и форм.


🛠 Технологии

	Backend			Django 6.0.3, Django REST Framework 3.15.2, Python 3.12
	API		    	REST API (JSON), DRF ViewSets, Serializers, Router
	Frontend		HTML5, CSS3, Bootstrap 5, JavaScript (Fetch API)
	База данных		SQLite3 (реляционная)
	ORM		    	Django ORM (select_related, фильтрация, ForeignKey)
	Инструменты		pip, venv, Git

📡 API Endpoints

	Метод	 	URL	                     	Описание
	GET	     	/api/transactions/	       	Список транзакций
	POST	 	/api/transactions/	       	Создание транзакции
	GET	     	/api/transactions/{id}/	 	Детали транзакции
	PUT	     	/api/transactions/{id}/	 	Обновление транзакции
	DELETE	 	/api/transactions/{id}/	 	Удаление транзакции
	GET	     	/api/statuses/	             	Список статусов

✅ Требования

    	- Django ORM используется
    	- Django REST Framework настроен
    	- Фильтрация списка операций по дате, статусу, типу операции, категории и подкатегории (query-параметры в API)
    	- Управление справочниками
    	- Логические зависимости (категория→тип, подкатегория→категория)
    	- Валидация данных
    	- Удобный интерфейс
